# Figma Design Process - Webbshop Mockup

**Grupp:** Grupp 8 - teamtröjor.se  
**Kurs:** Arbetsmetodik för utvecklare 2  
**Datum:** 2025-12-18

---

## Översikt

Detta dokument beskriver designprocessen för webbshop-mockupen i Figma. Vi använde Figma som huvudverktyg för kollaborativ design där alla i gruppen kunde arbeta i samma fil samtidigt, se varandras ändringar live, skapa återanvändbara komponenter, och bygga interaktiva prototyper.

---

## Designmetodik

### 1. Globala Komponenter Först

Vi började tillsammans med att skapa de globala komponenterna som skulle återanvändas på alla sidor:
- **Header/Navigation** - Med logo, menyalternativ och ikoner (varukorg, användare)

Detta var viktigt eftersom dessa komponenter skulle vara konsekventa över hela webbplatsen och spara tid när vi byggde de individuella sidorna. Genom att börja med globala komponenter tillsammans säkerställde vi att alla var med på samma designbeslut från början.

### 2. Bygga Sidor som Sektioner

Efter att de globala komponenterna var klara, byggde varje gruppmedlem sina sidor som individuella sektioner i Figma. Varje sida designades som en komplett vy med alla nödvändiga element baserat på user stories och personas.

---

## Sidor som Skapades

Totalt skapades **9 sidor** i Figma:

### Publika Sidor

**1. Startsida (Publik)**
- Överblick på kampanjvaror med extrapriser
- Utvalda produkter
- Hero-sektion med huvudbild och call-to-action
- Global header

**2. Shop-sida (Publik)**
- Kategorier (fotbollslag) med produkter
- Produktlista/grid med filteralternativ
- Sorteringsfunktioner
- Produktkort med bild, namn, pris
- Global header

**3. Produktdetaljsida (Publik)**
- Detaljerad produktinformation
- Stor produktbild
- Beskrivning, storlekar, pris
- "Lägg i varukorg"-knapp
- Relaterade produkter
- Global header

**4. Varukorgssida (Publik)**
- Lista över produkter i varukorgen
- Kvantitetsreglage och möjlighet att ta bort produkter
- Totalpris
- Global header

**5. Checkout-sida (Publik)**
- Leveransinformation formulär
- Fraktalternativ med priser
- Betalningsmetod (Klarna, Swish, kortbetalning)
- Ordersammanfattning
- "Slutför köp"-knapp
- Global header

**6. Inloggningssida (Publik) - Hosein**
- Inloggningsformulär med e-post och lösenord
- Demo-användare för olika roller (kund, admin, ägare)
- Länk till registrering
- Global header

### Skyddade Sidor

**7. Admin-sida (Skyddad - Sara)**
- Dashboard med statistik
- Lägga till nya produkter
- Redigera produkter (namn, text, bild, pris, lagersaldo)
- Välja ut varor till utförsäljning
- Global header

**8. Butiksägarens sida (Skyddad - Oliver)**
- Översikt över försäljning
- Total försäljning
- Försäljning per dag
- Försäljning per kategori/lag
- Produkter som är slut i lager
- Kategorier som säljer bäst per lag
- Statistik och rapporter
- Global header

**9. Customer history page (Köphistorik) (Skyddad)**
- Användarens köphistorik och tidigare ordrar
- Lista över alla tidigare köp med ordernummer
- Orderstatus (väntar, bearbetas, skickad, levererad)
- Orderdatum och spårningsnummer
- Produkter som köpts med bild, namn, storlek, kvantitet och pris
- Leveransadress
- Totalpris per order
- Möjlighet att se produkt och spåra leverans
- Global header

---

## Designprinciper

- **Konsistens:** Alla sidor använder samma globala header för enhetlig upplevelse
- **Återanvändbara komponenter:** Knappar, formulärfält och kort designades som komponenter för att spara tid och säkerställa konsistens
- **Användarcentrerad design:** Designen baserades på personas (Oliver, Sara, Johan) och user stories
- **Responsiv design:** Designen tog hänsyn till olika skärmstorlekar

---

## Fördelar med denna Metod

1. **Tidsbesparing:** Genom att skapa globala komponenter först slapp vi upprepa arbete
2. **Konsistens:** Alla sidor fick samma look and feel genom återanvändbara komponenter
3. **Effektivitet:** När globala delar var klara gick det snabbt att bygga nya sidor
4. **Översikt:** Genom att ha alla sidor som sektioner fick vi en tydlig översikt över hela webbplatsen
5. **Kollaborativt arbete:** Figma möjliggjorde att alla kunde arbeta samtidigt och se varandras ändringar live
6. **Interaktiva prototyper:** Vi kunde skapa interaktiva prototyper för presentation och testning

---

## Kommunikation och Samarbete

Designprocessen stöddes av:
- **Discord:** Live chatting och voice channels för snabb feedback och beslutsfattande
- **Screen sharing:** För att visa designförslag och diskutera lösningar
- **Regelbundna check-ins:** För att hålla alla uppdaterade om framsteg

---

## Slutsats

Genom att börja med globala komponenter tillsammans och sedan bygga sidor som individuella sektioner skapade vi en konsistent och välstrukturerad webbshop-mockup i Figma. Processen visade på fördelarna med kollaborativ design och hur Figma kan användas effektivt för agil utveckling.
